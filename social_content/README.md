## Day-by-Day Breakdown

### Day 1: Ticket Detection (Issue #4)
**Problem:** Hunting through JIRA/Linear/GitHub for ticket context
**Hook:** "Ever read a PR and wonder 'what ticket is this even for?'"
**Value:** Context without the detective work

### Day 2: Title Enhancement (Issue #5)
**Problem:** Vague PR titles like "fix bug" and "update stuff"
**Hook:** "We've all done it. 'Fix bug.' 'Update stuff.'"
**Value:** Descriptive titles without manual effort

### Day 3: Custom Placeholders (Issue #6)
**Problem:** Generic templates that don't fit team workflows
**Hook:** "One-size-fits-all templates actually fit no one well"
**Value:** Your process, automated

### Day 4: Size Detection (Issue #7)
**Problem:** 3,000-line PRs that sit unreviewed for days
**Hook:** "Your heart sinks. This will take an hour to review properly."
**Value:** Visibility drives better PR hygiene
